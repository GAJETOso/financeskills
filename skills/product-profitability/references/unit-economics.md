# Unit Economics

## Define the Unit First
Per product sold, per order, per customer, per active user, per ride/delivery, per bbl, per policy. Mixed units = meaningless math. State the unit on every analysis.

## The Core Ledger (per unit)
```
Average revenue per unit
−  Variable COGS (materials, hosting, payment fees 1.5–3%, packaging)
−  Variable fulfillment (shipping, delivery, support cost per order)
=  Contribution margin 1 (CM1)
−  Direct marketing cost per acquired unit (CAC where unit = customer)
=  CM2 / fully variable profit
```
CM1 must be positive at maturity or growth multiplies losses. CM2 payback determines cash needs.

## Customer-Level Economics
- `LTV = CM1 per period × expected periods` (or `CM1 × 1/churn` for constant churn; discount if horizon > 2 yrs).
- LTV:CAC ≥ 3 healthy; CAC payback < 12–18 months B2B, < 6–12 B2C.
- Cohort discipline: compute by acquisition cohort and channel — blended numbers hide dying channels.

## Margin Bridge Habit
When unit margin moves, decompose: price effect + mix effect + input cost effect + efficiency effect. (Pairs with variance-analysis skill.)

## Traps
1. Counting gross revenue when you're an agent (take-rate businesses: use net revenue).
2. Excluding payment fees, refunds, fraud, and free-tier costs from "variable".
3. LTV on projected retention with < 12 months of data.
4. Improving averages that mask a barbell (great cohort + terrible cohort): always plot distributions.
5. Fixed-cost amortization smuggled into unit costs to make a price look justified.
